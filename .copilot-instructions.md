# Copilot Instructions

## Running commands

- Prefer using `make` targets when the project has a `Makefile`.
- Before running tests or linters, read the project's `README` (or `README.md`) to find the correct commands.

## Git

- When committing, always use `git commit -e` (or `-em` with a message) so the editor opens for review.
- When running `git commit -e` or `git commit -em`, set `initial_wait` to at least 300 seconds (5 minutes) to allow time for editing the commit message.
- When generating a commit message, prefix it with the ticket number extracted from the branch name (e.g. branch `ABC-123-fix-bug` → prefix `[ABC-123]`). Extract the ticket by matching the pattern `[A-Z]+-[0-9]+` from the branch name. Do **not** add a prefix if the branch is `main` or `master`.

## Docker

- If you receive Docker errors when running tests (e.g. port conflicts, container issues), kill existing containers first:
  ```bash
  docker kill $(docker ps -q)
  ```
  Then retry the command.
